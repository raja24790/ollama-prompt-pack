# Business, Sales & Strategy Prompts

## Strategy & Planning

### 1. Business Plan Section Writer
```
Write the {section} section of a business plan for {business description}.

Sections: Executive Summary / Market Analysis / Competitive Landscape / Revenue Model / Marketing Strategy / Operations Plan / Financial Projections / Team

Requirements:
- Use specific numbers and market data where possible
- Be realistic (investors have seen 1000 hockey stick projections)
- Address risks honestly
- Include 3-year timeline with milestones
- Keep it concise — every sentence must earn its place
```

### 2. Pricing Strategy Developer
```
Develop a pricing strategy for {product/service}.

Context:
- Target market: {describe}
- Competitors charge: {price range}
- Our costs: {if known}
- Value delivered: {describe the outcome for customer}

Analyze:
1. Value-based pricing (what's it worth to the customer)
2. Competitive positioning (premium/par/economy)
3. Pricing psychology (anchoring, decoy effect, charm pricing)
4. Tier structure (3-tier recommendation with features per tier)
5. Launch pricing vs mature pricing
6. Discount strategy (when/how much)
```

### 3. Go-to-Market Plan
```
Create a go-to-market plan for launching {product} to {target market}.

Include:
1. **Target customer profile** (who, where, what they care about)
2. **Positioning statement** (for [audience] who [need], [product] is [category] that [benefit])
3. **Channel strategy** (ranked by expected ROI)
4. **Launch timeline** (pre-launch, launch week, post-launch)
5. **Success metrics** (what to measure in first 30/60/90 days)
6. **Budget allocation** (% across channels)
7. **Risk mitigation** (what if launch underperforms)
```

### 4. OKR/Goal Writer
```
Write quarterly OKRs for {team/company} focused on {priority area}.

Format:
**Objective:** {Qualitative, inspiring goal}
- KR1: {Quantitative, measurable, time-bound}
- KR2: {Different dimension of the objective}
- KR3: {Quality/health metric to prevent gaming}

Provide 3 objectives with 3 key results each.
Each KR should be: specific, measurable, achievable but stretchy, and independently valuable.
```

### 5. Competitive Positioning Map
```
Map the competitive landscape for {market/product category}.

Create:
1. **Key players:** List 5-8 competitors with brief description
2. **2x2 matrix:** Choose the two dimensions that matter most to buyers (e.g., price vs features, simplicity vs power)
3. **Positioning for each player** on the matrix with reasoning
4. **White space:** Where is there an underserved position
5. **Our position:** Where should we be and why
6. **Defensibility:** What makes our position hard to copy
```

## Sales

### 6. Cold Outreach Sequence
```
Create a {N}-touchpoint outreach sequence to {target persona} about {product/value prop}.

For each touchpoint:
- Channel (email/LinkedIn/phone)
- Timing (days after previous touch)
- Subject/opening
- Message (under 100 words for email, under 300 chars for LinkedIn)
- CTA (one clear action)

Angle progression: helpful → credible → relevant → direct ask
Total sequence: 2-3 weeks max. After no response: graceful close.
```

### 7. Objection Handler
```
Prepare responses for these common sales objections for {product}:

1. "It's too expensive"
2. "We're using {competitor}"
3. "We need to think about it"
4. "We don't have budget right now"
5. "Can you send me some information?"
6. "We tried something similar before and it didn't work"
7. "I need to check with my boss"
8. "We'll revisit next quarter"

For each: acknowledge → reframe → proof point → question back.
Keep responses conversational, not scripted. 2-3 sentences max.
```

### 8. Sales Email Templates
```
Write sales email templates for these scenarios:
1. First outreach (cold)
2. Follow-up after demo
3. After they went quiet (re-engagement)
4. Proposal follow-up
5. Closing email (create urgency)
6. Lost deal win-back (3 months later)

Each email: subject line + body under 150 words + clear CTA.
Product: {describe}
Target: {describe buyer persona}
```

### 9. Case Study Writer
```
Write a customer case study for {company} using {our product}.

Structure:
1. **Headline:** {Result} — {Company name}
2. **Quick stats:** 3 key metrics in boxes
3. **Challenge:** What problem they faced (use their words)
4. **Solution:** How they used our product (specific features)
5. **Results:** Quantified outcomes with before/after
6. **Quote:** A compelling testimonial
7. **About:** Brief company description

Information:
{provide any available details about the customer}
```

### 10. Discovery Call Script
```
Create a discovery call framework for selling {product} to {persona}.

Structure (30 minutes):
1. **Rapport** (2 min): Opening, agenda setting
2. **Situation** (5 min): Understand their current state
3. **Problem** (10 min): Deep dive into pain points
4. **Impact** (5 min): Quantify the cost of the problem
5. **Vision** (5 min): Paint the after picture
6. **Next steps** (3 min): Concrete next action

For each section: provide 3-4 specific questions to ask.
Include: what to listen for, red flags, and when to pivot.
```

### 12. FAQ Generator
```
Generate 15 FAQs for {product/service}. Organize into categories:

**Getting Started** (3-4 questions)
**Features & Usage** (4-5 questions)
**Pricing & Billing** (3-4 questions)
```

## Project Management

### 13. Project Kickoff Document
```
Create a project kickoff document for {project description}.

Include:
1. **Project overview:** What and why (3 sentences)
2. **Goals:** 3 measurable objectives
3. **Scope:** What's in / what's explicitly out
4. **Timeline:** Key milestones with dates
5. **Team:** Roles and responsibilities (RACI if appropriate)
6. **Risks:** Top 5 risks with mitigation
7. **Communication plan:** Meeting cadence, channels, escalation
8. **Definition of done:** How we know it's complete
```

### 14. Retrospective Facilitator
```
Facilitate a sprint retrospective. Team context: {describe recent sprint/period}.

Structure:
1. **What went well** (celebrate wins — list 5+ items)
2. **What didn't go well** (no blame — identify patterns)
3. **Root causes** (for the top 2 issues, dig deeper)
4. **Action items** (specific, owned, time-bound — max 3)
5. **Experiments** (one thing to try differently next sprint)

If the team keeps having the same retrospective items, call it out and suggest systemic fixes.
```

### 15. Status Report Writer
```
Write a {weekly/monthly} status report for {project/team}.

Information:
{paste raw notes, metrics, updates}

Format:
1. **Summary** (3 sentences — headline, progress, outlook)
2. **Key metrics** (table with target vs actual)
3. **Completed this period** (bullet list, linked to goals)
4. **In progress** (with % complete and blockers)
5. **Risks & issues** (red/yellow/green with mitigation)
6. **Next period priorities** (top 3-5)
7. **Decisions needed** (from stakeholders)

Keep it scannable. Leadership reads bullet points, not paragraphs.
```
