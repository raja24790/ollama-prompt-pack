# Content Writing & Communication Prompts

## Blog & Article Writing

### 1. SEO Blog Post Writer
```
Write a {word count}-word blog post about "{topic}". Requirements:
- Target keyword: "{keyword}" (use in title, H1, first paragraph, 2-3 H2s)
- Structure: Hook → Problem → Solution → Steps → FAQ
- Write for {audience level: beginner/intermediate/expert}
- Include 3-5 H2 headings and 2-3 H3 subheadings
- Natural keyword density (~1.5%)
- End with a clear CTA
- Add 3 FAQ questions at the bottom
- Tone: {conversational/professional/authoritative}
```

### 2. Listicle Generator
```
Write a "Top {N} {topic}" article. For each item include:
- A catchy name/title
- 2-3 sentence description of what it is
- Key benefit (why it matters)
- One concrete example or data point
- Verdict line (who it's best for)

Intro: 2-3 sentences setting context. Conclusion: summary comparison table.
```

### 3. How-To Guide Writer
```
Write a step-by-step guide: "How to {task}". Structure:
- **Intro:** What they'll achieve and time estimate
- **Prerequisites:** What they need before starting
- **Steps:** Numbered, each with a clear action verb start
- **Common mistakes:** 3 pitfalls to avoid
- **Troubleshooting:** 3 common issues + fixes
- **Next steps:** What to do after completing

Include code blocks/commands where relevant. Write at a {level} level.
```

### 4. Tutorial Writer
```
Write a technical tutorial for {topic}. Include:
- A concrete project to build by the end
- All code snippets (complete, copy-pasteable)
- Explain WHY not just HOW at each step
- Screenshots/terminal output descriptions where helpful
- "Checkpoint" boxes where the reader should test their progress
- Troubleshooting section for common errors
- Link suggestions to prerequisite knowledge
```

### 5. Comparison Article
```
Write a balanced comparison: "{A} vs {B}" for {audience}. Structure:
- Quick verdict (1-2 sentences up front)
- Overview of each (what it is, who makes it)
- Comparison table (features, pricing, pros/cons)
- Deep dive on 4-5 key comparison areas
- "Choose A if..." and "Choose B if..." sections
- Final recommendation

Be honest about weaknesses. Don't force a winner if it depends on use case.
```

## Email Writing

### 6. Cold Email Generator
```
Write a cold email to {recipient type} about {purpose}. Requirements:
- Subject line under 50 characters (3 options)
- Opening line that's personalized (use a hook about their company/role)
- Value proposition in 1-2 sentences
- Social proof (brief)
- Clear CTA (one specific action)
- Total length: under 150 words
- Tone: {friendly professional / direct / casual}
- P.S. line with urgency or bonus
```

### 7. Email Sequence Builder
```
Create a {N}-email nurture sequence for {product/goal}. For each email:
- Subject line (with open rate optimization)
- Preview text
- Email body (150-300 words)
- CTA
- Send timing (days between emails)

Sequence arc: Introduction → Value → Social proof → Objection handling → Offer → Last chance
```

### 8. Professional Email Templates
```
Write {N} email templates for these business scenarios:
{list scenarios — e.g., "follow up after meeting", "request introduction", "decline politely"}

Each template should:
- Be adaptable (use [brackets] for customizable parts)
- Include subject line
- Be concise (under 200 words)
- Have appropriate tone for business context
- Include a clear next step
```

## Marketing Copy

### 9. Landing Page Copy
```
Write landing page copy for {product/service}. Target audience: {describe}.

Sections needed:
- **Hero:** Headline (under 10 words) + subheadline + CTA
- **Problem:** 3 pain points (use "you" language)
- **Solution:** How the product solves each pain point
- **Features:** 6 features with benefit-focused descriptions
- **Social proof:** Testimonial templates + stats section
- **Pricing:** 3 tiers with anchor pricing psychology
- **FAQ:** 5 questions addressing objections
- **Final CTA:** Urgency + guarantee

Use power words. Focus on benefits over features. Create FOMO without being sleazy.
```

### 10. Product Description Writer
```
Write a product description for {product}. Include:
- Headline that hooks (benefit-first, not feature-first)
- 1-paragraph overview (what it is + who it's for)
- 5-8 bullet points (feature → benefit format: "X so that Y")
- Technical specs section
- Use case examples (3 scenarios)
- Comparison to alternatives (honest, not aggressive)

Tone: {premium/casual/technical/accessible}
Length: {short (100 words) / medium (250 words) / full (500+ words)}
```

### 11. Social Media Posts
```
Create {N} social media posts about {topic} for {platform}. For each:
- Platform: {Twitter/LinkedIn/Instagram}
- Character limit compliance
- Include a hook in the first line
- Add relevant hashtags
- Include a CTA (link/comment/share)
- Emoji usage: {none/minimal/normal}

Mix of types: educational, behind-the-scenes, question/engagement, promotional, story-based.
```

### 12. Ad Copy Generator
```
Write {N} ad variations for {product/service}. For each ad:
- Headline 1 (30 chars max)
- Headline 2 (30 chars max)
- Description (90 chars max)
- Display URL path

Use these angles: pain point, benefit, social proof, urgency, curiosity.
Target audience: {describe}
Platform: {Google Ads/Facebook/LinkedIn}
```

## Documentation

### 13. API Documentation Writer
```
Write API documentation for this endpoint. Include:
- Description of what it does
- Authentication requirements
- Request format (method, URL, headers, body with types)
- Response format (success + error responses with examples)
- Rate limiting info
- Code examples in Python, JavaScript, and curl
- Error codes table

Endpoint: {describe or paste code}
```

### 14. README Generator
```
Write a professional README.md for {project}. Include:
- Project name + one-line description
- Badges (build status, license, version)
- Screenshot/demo GIF placeholder
- Features list (bulleted)
- Quick start (3-5 steps)
- Installation (detailed, OS-specific)
- Usage examples (3 common use cases)
- Configuration options table
- Contributing guidelines
- License
- Credits/acknowledgments
```

### 15. Technical Specification
```
Write a technical specification for {feature/system}. Include:
- **Overview:** What and why (2-3 paragraphs)
- **Goals/Non-goals:** What this does and explicitly doesn't do
- **Design:** Architecture, data flow, API contracts
- **Alternatives considered:** 2-3 options with tradeoffs
- **Implementation plan:** Phases with milestones
- **Testing strategy:** Unit, integration, E2E
- **Rollout plan:** Feature flags, gradual rollout
- **Open questions:** Unresolved decisions
```
